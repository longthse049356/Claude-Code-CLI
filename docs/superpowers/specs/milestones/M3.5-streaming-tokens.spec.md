> Spec chi tiết cho implementation. Đọc design doc trước: `docs/superpowers/specs/2026-04-21-streaming-tokens-fix-design.md`.

## 3. File Specifications

### 3.1 `src/agent/progress.ts` — THÊM token store

**Responsibility:** Lưu accumulated text streaming per channel. Pattern song song với progress store hiện tại.

**Thêm vào cuối file (không touch code cũ):**

```typescript
// --- Token streaming store ---
// Accumulated assistant text streaming per channel.
// Worker appends; SSE endpoint polls delta via getTokensSince.

const tokenStore = new Map<string, string>();

export function appendToken(channelId: string, delta: string): void {
  const current = tokenStore.get(channelId) ?? "";
  tokenStore.set(channelId, current + delta);
}

export function getTokens(channelId: string): string {
  return tokenStore.get(channelId) ?? "";
}

/**
 * Returns text appended since `offset` (length-based cursor).
 * Returns empty string if no new text. Caller passes back `text.length` as next offset.
 */
export function getTokensSince(channelId: string, offset: number): string {
  const current = tokenStore.get(channelId) ?? "";
  if (offset >= current.length) return "";
  return current.slice(offset);
}

export function clearTokens(channelId: string): void {
  tokenStore.delete(channelId);
}
```

**Function signatures (chính xác — không đổi):**
- `appendToken(channelId: string, delta: string): void`
- `getTokens(channelId: string): string`
- `getTokensSince(channelId: string, offset: number): string`
- `clearTokens(channelId: string): void`

**Constraints:**
- Đặt **dưới** code progress hiện có. Không reorder, không refactor existing.
- Không thêm class. Không thêm options object. Functions thuần như progress.

### 3.4 `src/server/router.ts` — Emit token events trong poll loop

**Responsibility:** Mỗi poll tick, check token delta và emit `event: token` với delta text.

**Thay đổi 1:** Thêm import.

**Trước (`router.ts:26`):**
```typescript
import { getProgress } from "../agent/progress.ts";
```

**Sau:**
```typescript
import { getProgress, getTokensSince } from "../agent/progress.ts";
```

**Thay đổi 2:** Trong stream loop của POST `/channels/:id/messages/stream`, thêm token emission **giữa** progress check và message check.

**Trước (`router.ts:170-184`):**
```typescript
async start(controller) {
  const deadline = Date.now() + TIMEOUT_MS;
  let lastKeepalive = Date.now();
  let lastProgressJson = "";

  while (Date.now() < deadline) {
    if (req.signal.aborted) break;

    // Emit progress events when worker status changes
    const progress = getProgress(channelId);
    const progressJson = JSON.stringify(progress);
    if (progress && progressJson !== lastProgressJson) {
      controller.enqueue(encoder.encode(sseEvent("progress", progress)));
      lastProgressJson = progressJson;
    }

    const newMessages = getMessagesAfter(channelId, cursor);
    ...
  }
}
```

**Sau:**
```typescript
async start(controller) {
  const deadline = Date.now() + TIMEOUT_MS;
  let lastKeepalive = Date.now();
  let lastProgressJson = "";
  let lastTokenOffset = 0;  // ← NEW

  while (Date.now() < deadline) {
    if (req.signal.aborted) break;

    // Emit progress events when worker status changes
    const progress = getProgress(channelId);
    const progressJson = JSON.stringify(progress);
    if (progress && progressJson !== lastProgressJson) {
      controller.enqueue(encoder.encode(sseEvent("progress", progress)));
      lastProgressJson = progressJson;
    }

    // Emit token delta if new text streamed in                ← NEW BLOCK
    const tokenDelta = getTokensSince(channelId, lastTokenOffset);
    if (tokenDelta) {
      controller.enqueue(encoder.encode(sseEvent("token", { text: tokenDelta })));
      lastTokenOffset += tokenDelta.length;
    }

    const newMessages = getMessagesAfter(channelId, cursor);
    ...
  }
}
```

**Constraints:**
- KHÔNG đổi `POLL_INTERVAL_MS` (giữ 500ms).
- KHÔNG đổi `KEEPALIVE_INTERVAL_MS`, `TIMEOUT_MS`.
- KHÔNG reset `lastTokenOffset` khi tool call (tool sẽ trigger `clearTokens` ở worker, sau đó `getTokensSince(0+oldOffset, len=0)` trả "" → không emit gì → đúng).

  ⚠️ **Edge case quan trọng:** Khi worker `clearTokens` giữa turn (vì tool call), `tokenStore.get(channelId)` trả `undefined` → `getTokensSince` returns `""`. `lastTokenOffset` vẫn là giá trị cũ (vd: 200). Khi worker append text mới của tool round 2 (vd 50 char), `current.length = 50`, nhưng `offset = 200` → `offset >= current.length` → return `""`. **BUG.**

  **Fix:** Trong `getTokensSince`, nếu `offset > current.length` (text đã bị clear/reset) → reset offset implicitly bằng cách return từ đầu:

  ```typescript
  export function getTokensSince(channelId: string, offset: number): string {
    const current = tokenStore.get(channelId) ?? "";
    if (offset > current.length) {
      // Store was cleared/reset — return full current text, caller should reset offset
      return current;
    }
    if (offset >= current.length) return "";
    return current.slice(offset);
  }
  ```

  Và trong router, sau khi emit, set `lastTokenOffset` = current text length, KHÔNG cộng dồn:

  ```typescript
  const tokenDelta = getTokensSince(channelId, lastTokenOffset);
  if (tokenDelta) {
    controller.enqueue(encoder.encode(sseEvent("token", { text: tokenDelta })));
    lastTokenOffset = getTokens(channelId).length;  // ← reset to current absolute length
  }
  ```

  Cần thêm `getTokens` vào import: `import { getProgress, getTokens, getTokensSince } from "../agent/progress.ts";`

- KHÔNG emit "token" khi `tokenDelta` rỗng (đã có `if (tokenDelta)`).
- Token event PHẢI emit TRƯỚC "done" event (nếu cùng tick có cả delta và reply, FE cần thấy token trước rồi done sau — order trong code đã đúng vì done check sau token check).

## 4. SSE Event Format (reference)

Format đã tồn tại từ M3, không đổi. Spec lại để Sonnet đọc rõ:

```
event: progress
data: {"type":"thinking"}

event: token
data: {"text":"Hello"}

event: token
data: {"text":" world"}

event: progress
data: {"type":"tool_call","toolName":"read_file","iteration":1}

event: progress
data: {"type":"tool_done","toolName":"read_file"}

event: token
data: {"text":"The file contains..."}

event: done
data: {"message":{"id":"...","channel_id":"...","text":"Hello world\n\nThe file contains...","role":"assistant",...}}
```

**Quan trọng:**
- Mỗi `token` event chỉ chứa **delta** (text mới kể từ event trước), KHÔNG phải full accumulated text.
- FE cộng dồn (đã làm: `ChatPanel.tsx:98-104`).

## 5. Edge Cases & Error Handling

| Case | Behavior | File handle |
|---|---|---|
| LLM stream throw giữa chừng (vd 429) | `sendMessage` throw → `worker-loop.tick()` catch line 155 → log → KHÔNG emit done → router timeout 120s → emit "error" | `worker-loop.ts:155` |
| Client disconnect giữa stream | `req.signal.aborted` check ở line 176 → break loop → controller close. Worker tiếp tục chạy & save DB (không lãng phí LLM call) | `router.ts:176` |
| Server restart giữa turn | Token store mất (in-memory). Reply chưa save → mất hoàn toàn (acceptable cho M3.5). User retry. | N/A |
| `onToken` callback throw | Anthropic SDK swallow internally, stream tiếp tục. KHÔNG cần handle. | `anthropic.ts` |
| Tool call giữa turn | Worker `clearTokens` → token store rỗng → `getTokensSince` return "" → router không emit. Sau tool xong, worker `clearTokens` lại + sendMessage → text mới → router emit lại từ đầu. | `worker-loop.ts:123` |
| Multi tool iterations (5+ tool) | Mỗi iteration `clearTokens` + new `appendToken`. UI thấy text reset & restart sau mỗi tool round. **Acceptable** — tốt hơn không có gì. Lớn fix UX này. | N/A |
| Empty reply (LLM trả "") | `replyText.trim()` rỗng → skip create reply (`worker-loop.ts:138`) → router timeout. Tokens đã emit bị "mồ côi" trong UI draft. **Acceptable, edge case hiếm.** | N/A |
| 2 user message liên tiếp | Worker xử lý lần lượt (cursor advance). Token store chỉ giữ turn hiện tại. | N/A |

## 7. Acceptance Criteria

Verify bằng tay (không dùng test):

- [ ] **AC1 — Token chảy realtime:** Gõ "viết cho tôi 1 đoạn 500 chữ về Bun runtime". Chữ đầu tiên xuất hiện < 2s sau khi nhấn Enter.
- [ ] **AC2 — Tool call vẫn hoạt động:** Gõ "đọc file package.json và tóm tắt nội dung". UI hiển thị "Calling read_file…" → text tóm tắt chảy ra sau đó.
- [ ] **AC3 — Multi-tool turn:** Gõ "đọc file package.json, sau đó grep từ 'bun' trong src". UI hiển thị tuần tự các tool, sau đó text reply chảy ra.
- [ ] **AC4 — DB persistence:** Sau khi message hoàn tất, F5 refresh → cả user message và assistant reply đều còn.
- [ ] **AC5 — Test cũ pass:** `bun test` — tất cả pass, không có regression.
- [ ] **AC6 — Test mới pass:** Test A + Test B ở section 6.1 pass.
- [ ] **AC7 — UI không đụng:** `git diff main -- packages/ui/` rỗng.
- [ ] **AC8 — TypeScript strict:** `bun run typecheck` (hoặc `bunx tsc --noEmit`) pass.
- [ ] **AC9 — Connection drop graceful:** Trong lúc stream, Ctrl+C kill server. FE hiện error state, không crash. Restart server, F5 → message cũ load từ DB OK.
- [ ] **AC10 — No re-emit on stale offset:** Sau khi tool call clear tokens, text mới của round 2 hiển thị đúng (không trống, không trùng round 1).
